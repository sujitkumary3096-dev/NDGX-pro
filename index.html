The Vision — OmniGuard v3.0

Sovereign Security
for the Modern Web.

OmniGuard Terminal was built from a singular premise: every critical cryptographic operation should run locally, in your browser, with zero dependence on external servers, third-party APIs, or corporate infrastructure.

The Mandate

Modern infrastructure has been captured by a handful of corporations that charge rent on computation and architect systems to create dependency. OmniGuard is a direct rejection of this paradigm.

Every operation in this terminal — encryption, hashing, key generation, signing — executes via the browser's native Web Crypto API. Private keys and plaintext never leave your device.

Adversarial Design

Every feature is designed assuming an adversarial environment. Threat models include passive surveillance, active MITM attacks, social engineering vectors, and timing-based side-channel attacks.

The architecture provides defense-in-depth at every layer. Security through obscurity is a fallacy — we reject it entirely. The codebase is the documentation.

Data Collection

OmniGuard Terminal collects zero data. No personal information is transmitted, stored on external servers, or shared with third parties. All operations are performed exclusively within your browser session using client-side JavaScript and the Web Crypto API.

Cryptographic Operations

All encryption, hashing, key generation, and signing operations are performed by your browser's native Web Crypto API (SubtleCrypto). Private keys, passphrases, and plaintext data never leave your device. We have no technical capability to access them even if we wanted to.

Local Storage

The Mem Guard module stores secrets in JavaScript memory (RAM) only. No data is written to localStorage, sessionStorage, IndexedDB, or any persistent browser storage mechanism. All secrets are permanently destroyed when the browser tab or session is closed.

Network Activity

OmniGuard Terminal makes no outbound network requests except for loading Google Fonts and Tailwind CSS at page load. The P2P Nexus module uses WebRTC — connections are established peer-to-peer using STUN/TURN servers for NAT traversal only. No message content is routed through servers.

Cookies and Tracking

No cookies are set. No tracking pixels are embedded. No analytics scripts are loaded. No advertising networks are present. This terminal is built for operational utility, not monetization through surveillance.

Open Source Audit Rights

You have the right to inspect every line of this terminal's code. The single-file architecture ensures that no obfuscated, minified, or externally-loaded code can introduce undisclosed data collection. The source code is the privacy guarantee.

Immutability Clause

This privacy policy is immutable by design. Because the terminal is serverless and self-contained, there is no mechanism by which a server-side update could retroactively alter the behavior of a deployed instance. The version you deploy is the version you audit.

Compliance: Architecturally incapable of violating user privacy. Zero-server. Zero-telemetry. Zero-trust in third parties. The user holds sole custody of all cryptographic material.
